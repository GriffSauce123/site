Real Estate Project

+

During my senior year of high school, I landed an internship with my city, working on a commercial real estate project aimed at redesigning the downtown area. Initially tasked with manually gathering property data, I quickly realized the inefficiency of this process. Instead, I developed a Python script that automated the retrieval and analysis of hundreds of PDF files, extracting the necessary data in minutes instead of days. This tool drastically improved our workflow and allowed for more informed decisions in the city's redesign efforts, saving countless hours and enabling faster progress.

During my senior year in high school, I got an internship with the city in which I lived. I was brought onto a small commercial real estate project, aimed at updating and conglomerating data to make better decisions when redesigning the downtown area. Though this was through the IT pathway program, I had done a lot of work on business analytics and project management principles.

State of the Project:

Reliable, current, and accessible data for these buildings did not exist.
The assessor's database became the main source of data, but it was difficult and time-consuming to find important metrics.
A previous project member had created a spreadsheet to consolidate the collected data, including Property ID, Property Location, Previous or Current Tenant, Type of Business, and Owner. ( To be done by hand )
I was given a set of desired metrics to look for with each property.

My Responsibilities:

Initially, I was tasked with creating a project outline proposal, outlining the present state of the project, the goal of the project, and the steps to get there.
I interviewed government employees, commercial real estate agents, and members of the community in order to best determine the most vital metrics to track and make available.
I sought approval directly from the Mayor‚Äôs office, walking them through the project proposal and the timeline I had developed.
Once another intern was onboarded, I was tasked with filling them in, and assisting in the delegation of tasks.
We were to manually locate, clean, and upload the data for each property by hand from the assessors database PDF documents.

My Scheme:

Though we spent a week manually locating and filling in data, I was disheartened by the repetition, and the inefficiency of the whole operation.
I noticed that all the documents were stored in an S3 bucket with a convenient url structure, giving me the ability to locate each document by parcel id ( we were given the IDs of the properties of interest )
At this point I was essentially ignoring my manual data entry task, but writing a program that would pull each pdf document from the s3 bucket onto the machine doing the data analysis.
The next challenge was extracting meaningful data from the pdf, which was not set up in a machine-readable way. Data points were in different places, with slightly different labels, and everything was a mess.
Trial and error was how I spent the next few days. Scouring through the documents to find rough pixel coordinates where each metric was positioned.
I increased the tolerance until each metric contained the desired data as well as the label ( and usually a lot of fluff information as well ), and now I had to clean and filter the data in a more accessible format.
Since I was writing everything in Python, it was an incredibly easy choice to use the regular expression library ( regex ) in order to return only the values and the labels that were of interest.
Multi-line and more complex metrics required algorithms that looped through data points, and calculated a final result before returning anything. ( Anything that needed to be summed )
Once data was in a raw form, it was appended into a csv file automatically with the parcel ID as the primary key. SQL would have been too technical for those overseeing the project.

Result:

I spent approximately five days on this project, which slowed our progress significantly. My program, once finished, completed a full week's worth of manual entry in less than ten minutes. ( downloading hundreds of PDF files to a dusty office PC is not blisteringly fast )
My coworker no longer had a job to do, so we transitioned into taking up-to-date, clear, high quality photos for each roadside property on the list.
Now, a couple of years later, the results of this project are being utilized to redesign Main St. making sure that each property is not adversely affected due to reduced frontage, lot size, zoning, or visibility. Now that there is a reflective abundance of data, informed decisions can be made more efficiently.
Nobody overseeing the project had any expectation of automation, or internal tool development. There was a short period of time where we were scratching our heads as there was no more work to do. I am proud of my work on this project.

Unfamiliar with the Seattle area code, this turns out to be a Microsoft representative wanting to hear the whole story. He wants to know every detail and all use cases I could come up with for this bug. I spent an hour on the phone with this gentleman, being as detailed as 14 year old me could be. I emailed screenshots, pictures of my monitor, and step by step instructions.

In the moment this led to annoyance and frustration, but as a whole this marked a real improvement that I helped bring into the world. Hidden within the depths of Windows 10, only expert tomfoolers would discover, is a solution to a problem that I connected the dots on. And yes, I still got into that school machine.
